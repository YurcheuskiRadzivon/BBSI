Персональные метаданные

Отзыв (revoke)

Отзыв не удаляет узел выпуска из журнала — добавляется новая транзакция «отозван». После отзыва в полной проверке документ отмечается как не действующий (код document_revoked в warnings; сама цепочка остаётся «успешной»).

Справка: атаки и симуляция

Общее. Атаки 1 и 2 выполняются только в памяти: подмена видна при проверке, затем узел восстанавливается, в nodes.json остаётся исходная цепочка. Атака 3 не добавляет недопустимый узел — сервер отклоняет транзакцию.

Атака 1 — подделка DocumentHash. Для выбранного issue временно меняется хэш содержимого без пересчёта node_hash. Ожидаются ошибки вроде node_hash_mismatch. После сценария данные узла откатываются.

Атака 2 — подмена эмитента. В памяти меняется поле IssuerCountry, подпись ECDSA остаётся от старого ключа → bad_signature. Затем откат.

Атака 3 — незаконный revoke. Отзыв разрешён только эмитенту первого issue этого document_id. Запрос с «чужой страной» не записывается в граф (AddRevoke). Для уже записанных узлов тот же принцип даёт illegal_revoke.
